## Resolved Issues (from original audit + fix pass)

### Previously Critical — Now Fixed
- **Pricing drift risk** — Pricing is now externalized to `default_pricing.json` with `updated_at` field. `check_pricing_staleness()` warns if >30 days old. Shipped defaults copied to `~/.credclaude/pricing.json` on first run.
- **Silent failure patterns** — Broad exception handling replaced with narrowed exception types (`json.JSONDecodeError`), debug logging on all skip/failure paths, and type-safe config loading with validation.
- **PID lock race condition** — Replaced TOCTOU `os.kill()` check with atomic `fcntl.flock()` file locking.
- **Timezone inconsistency** — All `datetime.now()` calls in `limit_providers.py` unified through a `_now()` helper that returns timezone-aware datetimes. `_fmt_last_sync` in `app.py` updated to handle aware datetimes consistently.

### Previously High — Now Fixed
- **`_load_snapshot` silent bare except** — Added `as e` + `logger.debug()` for traceability.
- **Private field access across module boundaries** — `app.py` no longer accesses `self._provider._official._backoff_until`. Backoff info exposed via `get_backoff_remaining_minutes()` public method on `CompositeLimitProvider`.
- **Duplicate `_fmt_relative` computation** — Countdown computed once per update cycle, reused.
- **Config type validation missing** — `load_config()` now validates `billing_day` (int), `warn_at_pct` (int/float), and `plan_tier` (valid string) after loading, resetting to defaults on type mismatch.
- **launchd plist unused log paths** — Removed misleading `StandardOutPath`/`StandardErrorPath` keys from the plist (they captured nothing since `open -a` spawns a separate process). Added explanatory comment.
- **Unprotected `_update()` cycle** — Wrapped in try/except so a single provider failure cannot crash the menu bar app silently.

### Previously Medium — Now Fixed
- **`notifications_enabled` config key had no UI toggle** — Settings dialog now includes on/off toggle for notifications (added during refactor).
- **No automated tests** — 6 test modules with ~200 test cases covering billing, config, cost_engine, formatting, ingestion, and limit_providers.
- **Period scan re-parses all files every refresh** — The refactored architecture uses the OAuth API for session data, not per-tick JSONL scanning. File cache used where JSONL scanning still applies.
- **No version identifier** — Version single-sourced from `credclaude/__init__.py`, used by `pyproject.toml` (dynamic), `build_app.sh`, and `app.py` startup log.
- **Warn lock files accumulate** — `cleanup_old_warn_locks()` runs on startup, deletes `.warn_*` files older than 7 days.
- **`setup.py` duplicated `pyproject.toml`** — `setup.py` simplified to minimal shim; all metadata in `pyproject.toml`.
- **build_app.sh silent failure on moved repo** — Launcher script now checks venv existence and shows a user-facing error dialog if missing.

## Issues Found & Fixed (2026-03-21 patch session)

### Fixed
- **OAuth API rate limiting (429) on startup** — The app called `_update()` immediately in `__init__`, making an API call on every app start. Multiple restarts during install (e.g., iterating on `bash install.sh`) rapidly exhausted the API's rate-limit budget. Fixed by replacing the immediate call with a 5-second one-shot timer (`_startup_update`), giving the API rate-limit window a chance to recover between restarts. (`credclaude/app.py`)
- **Poll interval too aggressive** — `REFRESH_INTERVAL_SEC` was 10 seconds (360 calls/hour), which triggered sustained 429 rate limiting under normal operation. Increased to 60 seconds (60 calls/hour). The in-memory cache (`CACHE_TTL_SEC = 60`) was already aligned to this — the mismatch was only with the timer. (`credclaude/config.py`)

### Notes on Rate Limiting Behaviour (for future reference)
- The 429 rate limit is **server-side** from `api.anthropic.com/api/oauth/usage` — it is not generated by our own code.
- Our backoff (`RATE_LIMIT_BACKOFF_STEPS_SEC = (120, 300, 600)` seconds in `OfficialLimitProvider`) is a *response* to 429s, not their cause. A parallel 5-min cooldown (`TOKEN_EXPIRED_COOLDOWN_SEC = 300`) applies to 401 (token expired) responses.
- Clicking **"Refresh Now"** calls `force_refresh()` which clears the local retry guard (`_retry_after`, `_retry_reason`) and fires a real API call immediately — bypassing any active cooldown. However, `_rate_limit_step` is **not** reset: if the server window hasn't cleared and another 429 arrives, the backoff step escalates from its current position (e.g., 2m → 5m). Refresh Now is intended for deliberate manual overrides; avoid hammering it repeatedly during an active 429 window.
- Querying `/api/oauth/usage` does **not** consume Claude AI quota. It is a read-only monitoring endpoint — equivalent to checking the "Plan usage limits" bar on claude.ai. Only prompts sent to Claude AI models consume the 5-hour session limit.

## Remaining Known Limitations

### Low
- **`estimate_five_hour_window` has no file cache** — This function re-reads all JSONL files on each call. However, it is NOT in the hot path (the OAuth API provides session data directly). Only matters if the official API is permanently unavailable and the estimator fallback is extended to use it.
- **Billing day fallback silently shifts period start** — `billing_period_start()` falls back to day 1 when `billing_day` doesn't exist in a month (e.g., day 31 in February). This is mathematically correct for billing alignment but could confuse users with non-standard billing days. No user-visible warning.
- **`_normalize_utilization` ambiguity at value 1.0** — API value 1.0 is interpreted as 100% (fraction format), not 1% (percent format). This matches the current API contract. If the API changes format, this threshold would need updating. Documented in code comments.
- **Naive datetimes in billing module** — `billing.py` uses naive `datetime.now()` intentionally — billing resets are local-date events. Consistent internally but differs from the timezone-aware convention in `limit_providers.py`.
- **`monitor.py` deprecated file still in repo** — The original single-file version is marked deprecated and kept for reference. Can be deleted.

### Architectural Notes
- **App tied to source repo path** — The `.app` bundle's launcher script has the repo path baked in at build time. If the source repo moves, the user must re-run `install.sh`. The launcher now shows an error dialog instead of failing silently.
- **OAuth token refresh is automatic** — CredClaude refreshes the token proactively (10 min before expiry) and reactively (on 401). Manual `claude auth login` is only needed when the refresh token itself expires (typically weeks/months).
- **Weekly cap tracking is API-dependent** — Weekly utilization is now shown when `seven_day.utilization` is present in the OAuth response. For accounts without a weekly cap (e.g. student/edu), this field is absent and the section is hidden. No local computation of weekly cap is performed.
